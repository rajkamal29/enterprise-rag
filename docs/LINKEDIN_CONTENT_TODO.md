## Reusable post template

1. Hook: One high-impact statement from real project experience.
2. Problem: What architecture decision or failure point appeared.
3. Approach: How it was solved (design and implementation).
4. Tradeoff: What was gained and what was sacrificed.
5. Evidence: Test count, deployment check, latency/SLO, or CI gate result.
6. CTA: Invite discussion from architects/recruiters.

## Evidence checklist to include in posts

- [ ] Number of tests passing
- [ ] CI quality gates used
- [ ] Deployment validation proof (/health and /ask)
- [ ] Security controls (managed identity, APIM policy)
- [ ] Performance or latency budget reference
- [ ] Final decision rationale (why this option)
